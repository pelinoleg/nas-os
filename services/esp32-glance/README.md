# esp32-glance — внешний экран статуса NAS

ESP32 + TFT-дисплей, который постоянно показывает состояние NAS: общий вердикт,
плитки (пул, бэкап, температура, …) и полоску доступности за 24 часа.
Вся логика на сервере: панель (Настройки → Экран) решает, какие плитки и в каком
порядке отдавать, — прошивку перезаливать не нужно.

## Протокол

`GET http://<nas>/api/glance?token=<токен>&lang=en&seq=<последний seq>`

- **token** — read-only токен из вкладки «Экран» (Настройки). Даёт доступ только
  к этим данным, управлять NAS им нельзя.
- **lang** — `en` (латиница, дефолтные шрифты TFT) или `ru`.
- **seq** — номер последнего полученного снимка; если ничего не изменилось,
  сервер отвечает `304` без тела — экран не перерисовывается.

Ответ `200`:

```json
{
  "v": 1, "host": "pi5", "status": "ok", "problems": [],
  "tiles": [
    {"id": "pool", "label": "Pool", "value": "1.1", "unit": "TB free", "state": "ok"}
  ],
  "avail": {"bars": [2,2,1,0,2, "..."], "pct24": 99.2},
  "seq": 42, "ts": 1783755743
}
```

`state` ∈ `ok|warn|danger`; `avail.bars` — 96 слотов за 24 ч: `2`=онлайн,
`1`=жив без сети, `0`=выключен, `-1`=нет данных.

## Сборка

1. Arduino IDE / PlatformIO, плата ESP32.
2. Библиотеки: **TFT_eSPI** (настроить `User_Setup.h` под свой дисплей —
   проверялось с ST7789 и ILI9341 240×320) и **ArduinoJson** (v7).
3. В `esp32-glance.ino` заполнить `WIFI_SSID`, `WIFI_PASS`, `NAS_HOST`, `TOKEN`.
4. Прошить. Экран сам переживает потери Wi-Fi и рестарты NAS
   (показывает «NAS UNREACHABLE», пока связь не вернётся).

## Идеи для доработки

- e-ink вместо TFT: опрашивать раз в минуту и перерисовывать только при смене `seq`;
- пищалка/светодиод при `status != ok`;
- кнопка — переключение страниц (плитки / графики истории).

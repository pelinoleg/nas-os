# esp32-glance — внешний экран статуса NAS

ESP32 + TFT-дисплей, который постоянно показывает состояние NAS: общий вердикт,
страницы плиток (пул, бэкапы, температура, …) и полоску доступности за 24 часа.
Вся логика на сервере: панель (Настройки → Экран) решает, какие страницы, плитки,
размеры и порядок отдавать, — прошивку перезаливать не нужно.

## Целевое железо

- **LilyGO T-Display-S3** (170×320, ST7789) — работает из коробки: в TFT_eSPI
  выбрать готовый `Setup206_LilyGo_T_Display_S3.h`. Кнопки BOOT/KEY листают
  страницы, плюс автоперелистывание (`PAGE_ROTATE_MS`).
- **LilyGO T-Display-S3 Long** (640×180, AXS15231B) — TFT_eSPI этот контроллер
  НЕ поддерживает: возьмите библиотеку из
  [Xinyuan-LilyGO/T-Display-S3-Long](https://github.com/Xinyuan-LilyGO/T-Display-S3-Long)
  (папку `lib` — в библиотеки Arduino) или Arduino_GFX. В скетче все обращения
  к дисплею собраны в четырёх функциях-примитивах (`drawTile`/`drawSpark`/
  `drawAvail`/`drawOffline` + `fillScreen`) — замените вызовы `tft.*` на
  аналоги выбранной библиотеки, остальное не трогается. Раскладка сама
  переключается на 3 колонки при ширине ≥480 px.

## Протокол

`GET http://<nas>/api/glance?token=<токен>&lang=en&seq=<последний seq>[&screen=<id>]`

**Несколько экранов**: в конструкторе (панель → Настройки → Экран → «Открыть
конструктор») можно завести до 4 «экранов» — у каждого свой пресет размера,
свои страницы и плитки. Устройство выбирает свой профиль параметром
`screen=<id>` (id показан в конструкторе, в скетче — константа `SCREEN_ID`);
без параметра отдаётся первый.

- **token** — read-only токен из вкладки «Экран» (Настройки). Даёт доступ только
  к этим данным, управлять NAS им нельзя.
- **lang** — `en` (латиница, дефолтные шрифты TFT) или `ru`.
- **seq** — номер последнего полученного снимка; если ничего не изменилось,
  сервер отвечает `304` без тела — экран не перерисовывается.

Ответ `200`:

```json
{
  "v": 2, "host": "pi5", "status": "ok", "problems": [],
  "pages": [
    {"name": "Главная", "tiles": [
      {"id": "pool", "label": "Pool", "value": "1.1", "unit": "TB free",
       "state": "ok", "size": "l"},
      {"id": "cputemp", "label": "CPU temp", "value": "58", "unit": "°C",
       "state": "ok", "size": "m", "spark": [52, 54, 61, "…48 точек за 24 ч"]}
    ]},
    {"name": "Бэкапы", "tiles": ["…"]}
  ],
  "tiles": ["…копия первой страницы — для старых прошивок…"],
  "avail": {"bars": [2,2,1,0,2, "..."], "pct24": 99.2},
  "seq": 42, "ts": 1783755743
}
```

`size` ∈ `s|m|l`: `l` — карточка на всю ширину с крупным значением, `m` —
карточка (2 колонки, на длинном экране 3), `s` — компактная строка.
Карточка: заголовок в верхнем углу, статус-точка справа, значение крупно по
центру; длинные строки автоматически уменьшаются/обрезаются.

**Стили и свободный режим.** У экрана есть `mode` (`flow` — сетка, `free` —
свободная расстановка) и `gap` (отступ между плитками сетки). В free-режиме
каждая плитка несёт `x/y/w/h` (в пикселях экрана, наложения разрешены — рисуйте
в порядке массива). Опциональный `st` — стиль плитки:

| ключ | что это | значения |
|------|---------|----------|
| `lp`/`vp`/`up` | позиция заголовка / значения / единицы | 9-сетка `tl…br`, `hide`; для единицы ещё `val` (приклеена к значению) |
| `ls`/`vs`/`us` | размер шрифта, px | 6…120 (скетч подбирает ближайший шрифт TFT) |
| `bg` | фон | нет ключа = карточка, `none`, `#RRGGBB` |
| `bw`/`bc` | рамка: толщина и цвет | 0…6 (0 = без рамки), `#RRGGBB` |

Всё это настраивается визуально в конструкторе (инспектор справа при выборе
плитки); скетч интерпретирует стиль сам — прошивку менять не нужно.

## Свой интерфейс поверх API

`value`/`unit` — готовые строки для отображения, а в каждом тайле есть **`raw`**
— машиночитаемые данные (байты, проценты, °C, epoch-времена):

```json
{"id": "pool", "value": "1.1", "unit": "TB free", "state": "ok",
 "raw": {"free": 1258291200000, "used": 157286400000, "pct": 12.1}}
```

Так что можно игнорировать нашу вёрстку целиком и строить свою: опрашивайте
`/api/glance?token=…` и берите `tiles[].raw`, `avail.bars`, `problems`,
`status`. Формат стабильный (`v: 2`), поле `seq`+HTTP 304 экономят перерисовки.

`state` ∈ `ok|warn|danger`; `avail.bars` — 96 слотов за 24 ч: `2`=онлайн,
`1`=жив без сети, `0`=выключен, `-1`=нет данных. У части плиток (температура,
CPU, память, скорость сети, пул) есть `spark` — 48 точек истории за сутки,
скетч рисует их мини-графиком в углу плитки.

Плитки настраиваются в панели: Настройки → Экран. Там же можно добавить свои
проверки: исполняемый скрипт в `~/nas-config/scripts/glance/`, первая строка
вывода `ok|warn|danger текст` — станет плиткой и влияет на общий `status`.

## Прошивка шаг за шагом (T-Display-S3)

Экран у платы впаян — ничего подключать не надо, только кабель USB-C.

1. **Arduino IDE** (arduino.cc). В *File → Preferences → Additional boards
   manager URLs* добавить:
   `https://espressif.github.io/arduino-esp32/package_esp32_index.json`,
   затем *Boards Manager* → установить **esp32 by Espressif**.
2. *Tools → Board* → **LilyGo T-Display-S3**. Если такой нет — **ESP32S3 Dev
   Module** и руками: *USB CDC On Boot: Enabled*, *Flash Size: 16MB*,
   *PSRAM: OPI PSRAM*, *Partition Scheme: 16M Flash (3MB APP)*.
3. *Library Manager* → установить **TFT_eSPI** (Bodmer) и **ArduinoJson** (≥7).
4. Настроить TFT_eSPI под панель: открыть
   `Документы/Arduino/libraries/TFT_eSPI/User_Setup_Select.h`, закомментировать
   `#include <User_Setup.h>` и раскомментировать строку с
   **`Setup206_LilyGo_T_Display_S3.h`** — профиль уже в комплекте библиотеки.
5. Открыть `esp32-glance.ino`, заполнить четыре константы сверху:
   `WIFI_SSID`, `WIFI_PASS`, `NAS_HOST` (IP NAS-а) и `TOKEN` — токен создаётся
   в панели: **Настройки → Экран → Токен → Создать**, там же «Копировать URL»
   (токен — это часть `token=…`). Для второго экрана впишите его id в
   `SCREEN_ID` (виден в конструкторе как `screen=…`).
6. Подключить плату по USB-C, выбрать появившийся порт (*Tools → Port*) и
   нажать **Upload**. Если порт не появился: зажать кнопку **BOOT**, нажать
   **RST**, отпустить BOOT — плата войдёт в режим прошивки.
7. После прошивки экран сам подключится к Wi-Fi и покажет плитки. Кнопки
   BOOT/KEY листают страницы. Потери Wi-Fi/NAS переживает сам: «offline Nm»
   на последних данных, «NAS UNREACHABLE» — если данных ещё не было.

**T-Display-S3 Long** — та же процедура, но вместо TFT_eSPI взять библиотеку
из репозитория LilyGO (см. «Целевое железо» выше) и заменить draw-вызовы.

Если что-то не заводится — смотрите Serial Monitor (115200): скетч пишет,
что происходит.

## Идеи для доработки

- e-ink вместо TFT: опрашивать раз в минуту и перерисовывать только при смене `seq`;
- пищалка/светодиод при `status != ok`;
- кнопка — переключение страниц (плитки / графики истории).
